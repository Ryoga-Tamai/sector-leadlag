## 0. 改訂概要（v2 → v3 の変更点と理由）

v3 は v2 の構成（GitHub Actions ＋ Python ＋ LINE ＋ GitHub Pages）を踏襲しつつ、**「論文再現バックテストの結果が正しいか判断できない」という現状を構造的に解消する**ことを目的に再定義したものである。

| # | 変更点 | 理由 |
|---|---|---|
| C1 | **戦略リターンを Open-to-Close (OC) リターンで厳密に評価する**（v2 の「日本側 CC リターンで近似」を廃止） | 論文の仕組みは「米国 CC 情報 → 日本の翌日 OC を取る」。CC 近似は日本のオーバーナイトリターンを損益に混入させ、戦略が狙う情報伝播構造を汚染する。検収ゲートとして機能しない |
| C2 | **26 銘柄の固定ユニバースを採用**（米国 9：XLC・XLRE を除外、日本 17） | XLC は 2018 年、XLRE は 2015 年の上場で、論文の「Cfull を 2010–2014 で推定」は文字通りには再現不可能。固定ユニバースにすれば Cfull 期間・評価期間を論文に合わせられ、銘柄入替起因のバグが恒久的に消える |
| C3 | **検証・受け入れ基準セクションを新設**（IC・分位スプレッド・ルックアヘッド監査） | v2 は「最終 R/R を論文と比較」のみ。スカラ 1 個ではズレの原因を切り分けられない。シグナルの予測力そのものを検証する層を追加する |
| C4 | **データ仕様セクションを新設**（auto_adjust、リターン定義、取引日インターセクション、欠損方針） | v2 はデータ取得仕様が Notebook に埋もれており、暗黙の dropna で評価期間が意図せず短縮されるリスクがあった |
| C5 | **年率化係数を 252 に明記** | 論文の式(27)(28)は係数「12」（月次用）と記載されているが、日次戦略のため 252 が正しい（AR 23.79% は日次平均×252 で整合）。論文側の誤植と判断 |
| C6 | **開発環境を Python 3.11 に固定**（venv 再構築手順を追加） | ローカルが macOS 標準 Python 3.9（LibreSSL）で、requirements.txt の scikit-learn 漏れ・NotOpenSSLWarning が既発生。データ取得の不安定要因を先に除去する |
| C7 | 開発フローに **SECTION 1.5（環境再整備）** を挿入。SECTION 8 を「リポジトリ作成」→「リモート確認・README 更新」に変更 | SECTION 0–1 はコミット済み・GitHub リポジトリは既存のため |

## 1. プロジェクト概要

### 1.1 目的

論文の部分空間正則化付き PCA による日米業種リードラグ戦略を、個人投資家として実運用するための支援ツールを完全無料で構築する。米国市場クローズ後に GitHub Actions が自動でシグナルを生成し、LINE で通知することで、日本市場オープン時に手動で発注できる状態を毎営業日整える。

### 1.2 スコープ

- 対象ユーザー：自分一人（個人専用）
- 規制対応：不要（自分用ツールは金融商品取引法の投資助言業に該当しない）
- 取引執行：完全手動（証券会社アプリから自分で発注）
- 対象市場：日本市場の TOPIX-17 業種別 ETF（17 銘柄）
- 運用コスト：0 円/月

### 1.3 スコープ外

- 自動売買（証券会社 API 連携）
- 他ユーザー向けの提供・配信
- マルチアセット拡張（FX、コモディティ等）
- 高頻度取引・日中シグナル
- Fama–French / Carhart 等のファクター回帰（論文の検証手法だが、個人ツールには不要）

### 2.2 数理パラメータ（論文準拠）

| パラメータ | 値 | 説明 |
|---|---|---|
| L | 60 | 推定ウィンドウ長（共通営業日） |
| K | 3 | 主成分数 |
| K₀ | 3 | 事前部分空間次元 |
| λ | 0.9 | 正則化強度 |
| q | 0.3 | 上位/下位分位（17 × 0.3 = 5.1 → 上位 5・下位 5 銘柄） |
| Cfull 期間 | 2010-01-01 〜 2014-12-31 | 事前固有値推定用（26 銘柄全てが存在する期間） |
| 年率化係数 | **252** | 日次戦略のため。論文式(27)(28)の「12」は誤植と判断 |

### 2.3 事前部分空間 V₀（論文 3.1 節）

26 次元空間内の 3 つの直交ベクトル：

1. **グローバルファクター** v₁：全 26 銘柄に等しい重み（1）
2. **国スプレッドファクター** v₂：米国側 +1、日本側 −1 を v₁ に直交化
3. **シクリカル/ディフェンシブファクター** v₃：シクリカル +1、ディフェンシブ −1、中立 0 の符号ベクトルを v₁・v₂ に直交化

V₀ = [v₁, v₂, v₃]（列直交、shape = (26, 3)）。

### 2.4 リターン定義（v3 で厳密化）

**PCA 推定用：Close-to-Close リターン**

```
r_cc[i, t] = AdjClose[i, t] / AdjClose[i, t-1] - 1
```

配当・分割調整後の終値を用いる（yfinance は `auto_adjust=True` で取得）。

**戦略評価用：Open-to-Close リターン（日本側のみ）**

```
r_oc[j, t] = AdjClose[j, t] / AdjOpen[j, t] - 1
```

同一営業日 t の調整後始値と調整後終値の比。配当・分割調整はその日の OHLC 全体に同一倍率でかかるため、Close/Open の比では相殺され、`auto_adjust=True` の Open・Close をそのまま使ってよい。
**v2 の「日本側 CC リターンで近似」は廃止する。**

**シグナル入力用：米国側 Close-to-Close リターン**

時点 t の米国終値確定後に観測可能な情報として、米国業種の当日 CC リターン `r_cc[u, t]` を用いる。

### 2.5 計算フロー（時点 t、米国市場クローズ後）

```
1. 共通営業日ベースで推定ウィンドウ W_t = {t-60, ..., t-1} を取得
   （※ t は含まない。off-by-one 厳禁）
2. W_t 内の平均・標準偏差で各銘柄の CC リターンを標準化 → 標準化リターン行列 Z_t
3. W_t 内相関行列 C_t を構築
4. 正則化相関行列 C_t^reg = (1-λ) C_t + λ C0
5. 固有分解 C_t^reg = V_t Λ_t V_t^T、上位 K 固有ベクトル抽出
6. 米国/日本ブロックに分割 → V_U,t^(K) (9×K)、V_J,t^(K) (17×K)
7. 米国の当日 t の CC リターンを W_t の平均・標準偏差で標準化 → z_U,t
   （※ z_U,t の標準化に t 自身の統計量を使わない。ウィンドウ統計量を使う）
8. ファクタースコア f_t = V_U,t^(K)^T z_U,t
9. 日本側へ復元 → シグナル ẑ_J,t+1 = V_J,t^(K) f_t
10. ẑ_J,t+1 をスコアとして日本 17 銘柄をランキング
    → 上位 5 銘柄ロング、下位 5 銘柄ショート
```

### 2.6 ロングショート構築

- ロング：上位 5 銘柄、等ウェイト（各 +1/5）
- ショート：下位 5 銘柄、等ウェイト（各 −1/5）
- グロス・エクスポージャー = 2、ネット = 0
- 戦略リターン：R_{t+1} = Σ_j w_{j,t+1} · r_oc[j, t+1]
  （シグナルは時点 t に生成、リターンは次の共通営業日 t+1 に実現）

## 3. データ仕様（v3 新設）

### 3.1 データソース

- **MVP〜運用初期**：yfinance（`auto_adjust=True`、Open・Close 両方を取得）
- **本格運用（Phase 3）**：J-Quants Light（月 ¥1,650、JPX 公式）への切替を検討。本要件定義の範囲では yfinance を前提とする

### 3.2 取引日インターセクション

日米は取引時間帯が非同期で、祝日も異なる。論文の「共通に観測可能な営業日のみ使用」に従い、**日本側と米国側の両方が取引日である日付のみ**を採用する。

実装ルール：
- 全 26 銘柄の価格データを取得後、26 銘柄すべてが非欠損である日付のみ残す
- この共通営業日インデックス上で「t」「t+1」を定義する。t+1 は単に共通営業日インデックスの次の行
- **暗黙の全期間 dropna で評価期間が意図せず短縮されていないこと**を、処理後に必ず「採用期間：開始日〜終了日、営業日数 N」として標準出力に表示する

### 3.3 欠損・異常値方針

- 26 銘柄のいずれかが要求期間で全く取得できない場合は例外を投げて停止
- 部分的な単発欠損（連続 1〜2 営業日）は許容し、共通営業日インターセクションで除外される
- 連続 3 営業日以上の欠損が検出された場合は警告ログを出す
- 価格が 0 または負、リターンが ±50% を超える日は異常値として警告ログを出す（停止はしない）

### 3.4 キャッシュ

- 取得した価格データは `data/cache_prices.csv` にキャッシュ
- 再実行時、要求日付範囲がキャッシュに含まれていればキャッシュから読む
- yfinance 取得失敗時は最大 3 回、指数バックオフでリトライ

## 4. 検証・受け入れ基準（v3 新設）

セクション 2（論文再現バックテスト）の合否は、**論文の R/R に一致するか否かではなく、シグナルに予測力があることを多角的に確認できるか**で判定する。R/R の完全一致はデータソース・期間・ユニバース・年率化の差異により原理的に期待しない。

### 4.1 検証指標

| 指標 | 定義 | 合格ライン |
|---|---|---|
| **Information Coefficient (IC)** | 各営業日 t における、シグナル ẑ_J,t+1 と実現 OC リターン r_oc[·,t+1] の順位相関（Spearman）。全期間の平均と t 値 | 平均 IC が正、かつ t 値 > 2 |
| **分位スプレッド** | シグナルで日本 17 銘柄を分位（例：5 分位）に分け、各分位の平均翌日 OC リターンを算出 | 上位分位 > 下位分位、かつ概ね単調 |
| **ルックアヘッド監査（シフト）** | シグナル系列をさらに +1 営業日ずらして P/L 再計算 | パフォーマンスが大きく低下する（崩れなければリーク有） |
| **ルックアヘッド監査（シャッフル）** | 各日のシグナルを銘柄方向にランダムシャッフルして P/L 再計算 | パフォーマンスがほぼゼロになる |
| **R/R（参考）** | AR / RISK（年率化係数 252） | [1.0, 3.0] なら「データ・期間差を踏まえ論文と整合」。0.5 未満かつ上記検証が不合格ならバグ |

### 4.2 合否判定

**合格条件（すべて満たすこと）**：
- IC が正、t 値 > 2
- 分位スプレッドが単調（上位 > 下位）
- ルックアヘッド監査（シフト・シャッフル）の両方でパフォーマンスが崩れる
- 上記が満たされていれば、R/R が論文値と乖離していても合格とする

**不合格（セクション 2 内で修正、次に進まない）**：
- IC がゼロ近傍または負
- ルックアヘッド監査でパフォーマンスが崩れない（＝未来情報のリーク。最優先で原因調査）
- 分位スプレッドが非単調または逆転

### 4.3 論文値との乖離要因（文書化用）

セクション 2 完了時、論文値（AR 23.79%、RISK 10.70%、R/R 2.22、MDD 9.58%）との差を、以下の既知要因とともに Notebook の考察セルに記録する：

1. データソースの違い（論文のデータソースは非公開、本ツールは yfinance）
2. ユニバースの違い（v3 は米国 9 銘柄、論文は 11 銘柄）
3. 評価期間・共通営業日インターセクションの差
4. OC リターンの実装差（始値の調整方法等）

## 9. リスクと既知の制約

### 9.1 戦略の現実的リスク

- **取引コスト**：論文の AR は理論値。実運用では手数料・スプレッド・貸株料・税金で大きく減る可能性
- **流動性リスク**：TOPIX-17 ETF の一部は流動性が薄く、スリッページが大きくなる場合がある
- **MDD の悪化**：論文の MDD はサンプル期間内の値。実運用では市場体制の変化で悪化しうる
- **モデル劣化**：論文の戦略が将来も有効である保証はない。継続モニタリングが必要

### 9.2 v3 の設計判断による既知の乖離

- 米国ユニバースを 9 銘柄に絞ったため、論文（11 銘柄）の数値とは完全一致しない。これは「想定内・文書化済みのズレ」として扱う（§4.3）
- yfinance は非公式ラッパーで、まれに欠損・遅延が発生する。本格運用では J-Quants Light への切替を検討

### 9.3 規制面（将来の参考）

純粋に個人用途を維持する限り、規制上の問題はない。ただし以下は再検討が必要：家族・友人への継続的なシグナル提供、SNS・ブログでの定期公開、対価の受領、法人化。

### 9.4 環境面

- macOS 標準 Python 3.9（LibreSSL）は yfinance の HTTPS 接続でまれに失敗する。SECTION 1.5 で 3.11 へ移行することで予防する
